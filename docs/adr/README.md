# ADRs (Architecture Decision Records)

Este diretório contém **registros de decisões arquiteturais** do projeto.

## Por que ADR?

- Documentar **contexto**, **decisão** e **consequências**
- Facilitar alinhamento do time e onboarding
- Evitar retrabalho e “decisões perdidas” em chat

## Como criar um ADR

1. Crie um novo arquivo seguindo o padrão:
   - `docs/adr/NNNN-titulo-curto-em-kebab-case.md`
2. Use o template em `docs/adr/template.md`
3. Numere sequencialmente (`0001`, `0002`, …)

## Status

Use um destes:
- `Proposto`
- `Aceito`
- `Depreciado`
- `Substituído por ADR-XXXX`
